## 2. Information We Collect

### 2.1 Information You Provide Directly

#### For Non-Anonymous Reports:
- **Personal Information:**
  - Full name
  - Phone number
  - Email address
  - National ID number (optional)

#### For All Reports (Anonymous & Non-Anonymous):
- **Report Details:**
  - Crime category (theft, kidnapping, corruption, etc.)
  - Description of incident
  - Date and time of incident
  - Location description

- **Media Evidence:**
  - Photos (JPEG, PNG, etc.)
  - Videos (MP4, MOV, etc.)
  - Audio recordings (if applicable)

- **Location Data:**
  - GPS coordinates (latitude/longitude)
  - IP address
  - Approximate location based on IP

### 2.2 Information Collected Automatically

- **Device Information:**
  - Browser type and version
  - Operating system
  - Device model
  - Screen resolution

- **Usage Data:**
  - Pages visited
  - Time spent on system
  - Actions performed
  - Errors encountered

- **Technical Data:**
  - IP address
  - Session ID
  - Cookies (if enabled)

### 2.3 Blockchain and IPFS Data

- **Blockchain Records (Cardano):**
  - Evidence hash (cryptographic fingerprint)
  - Transaction hash
  - Block number
  - Timestamp

- **IPFS Records:**
  - Content Identifier (CID)
  - File size
  - Upload timestamp
  - Number of nodes storing content

| **Courts & Judicial Authorities** | Legal proceedings | Evidence as required by law |
| **Prosecutor's Office** | Criminal prosecution | Evidence as required by law |

### 4.2 We DO NOT Share Data With:
- ‚ùå Third-party advertisers
- ‚ùå Commercial companies
- ‚ùå Social media platforms
- ‚ùå Foreign governments (except through legal treaties)
- ‚ùå Unauthorized individuals

### 4.3 Public Blockchain Transparency

**Important Notice:**
- Evidence hashes are recorded on **Cardano blockchain** (public ledger)
- Blockchain transactions are **publicly visible** but do not contain personal information

Your identity remains protected unless you choose to include it in the report.

## 6. Data Storage and Security

### 6.1 Storage Locations

| Data Type | Storage Location | Duration |
|-----------|------------------|----------|
| **Report Details** | PostgreSQL Database (Rwanda) | Permanent |
| **Media Evidence** | IPFS (Global Network) | Permanent |
| **Blockchain Proof** | Cardano Blockchain | Permanent |
| **User Accounts** | PostgreSQL Database (Rwanda) | Until account deletion |
| **Session Data** | Redis Cache | 24 hours |

### 6.2 Security Measures

We implement:
- üîí **Encryption:** SSL/TLS for data in transit
- üîí **Hashing:** SHA-256 for password storage
- üîí **Access Control:** Role-based permissions
- üîí **Audit Logs:** All access is logged
- üîí **Firewall:** Network security measures
- üîí **Regular Backups:** Daily database backups
- üîí **Two-Factor Authentication:** For admin accounts

### 6.3 Data Retention

#### Active Reports:
- Retained **permanently** for legal and investigative purposes

#### Closed Reports:
- Archived but **never deleted**
- Blockchain/IPFS records are **immutable**

#### User Accounts:
- Admin accounts: Active until resignation/termination
- Public accounts: Can be deleted upon request (reports remain)

## 9. Children's Privacy

### 9.1 Age Restriction
- Users must be **18 years or older** to create accounts
- Minors may submit reports in emergencies

### 9.2 Parental Consent
- If a minor submits a report, parental/guardian consent is assumed in emergency situations
- Schools and guardians may submit reports on behalf of minors

### 9.3 Data Protection for Minors
- Extra care is taken to protect minors' identities
- Reports involving child victims are flagged for sensitive handling

## 10. International Data Transfers

### 10.1 IPFS Global Network
- Evidence is replicated across **1000+ nodes worldwide**
- Nodes may be located in: USA, Europe, Asia, Africa

### 10.2 Blockchain Network
- Cardano validators are located **globally**
- Transaction data is public but pseudonymous

### 10.3 Legal Protections
- Rwanda has data protection agreements with regional partners

## 11. Data Breach Notification

### 11.1 Our Commitment
In the event of a data breach, we will:
- Notify affected users within **72 hours**
- Report to **RURA** (regulatory authority)
- Take immediate action to secure the system
- Provide guidance on protective measures

### 11.2 What Constitutes a Breach
- Unauthorized access to user accounts
- Leak of personal information
- System compromise affecting data integrity

### 11.3 Blockchain/IPFS Resilience
- Blockchain and IPFS data cannot be "breached" in traditional sense
- Content is already distributed and verifiable
- Hacking one node does not compromise the network
